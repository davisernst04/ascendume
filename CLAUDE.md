## Known Constraints & Gotchas

### Vercel Blob
- The blob store is configured as **private** — always use `access: 'private'`, never `'public'`
- Always pass `addRandomSuffix: true` to avoid conflicts when the same filename is uploaded more than once
- `BLOB_READ_WRITE_TOKEN` must be set or the upload step is skipped (the parse route handles this gracefully — `originalFileUrl` is nullable)

### OpenAI Structured Output (`generateObject`)
- OpenAI's `response_format` structured output requires **every field** in `properties` to appear in `required`
- Never use `.default(...)` on Zod fields passed to `generateObject` — it makes them optional and removes them from `required`, causing a schema validation error
- Always use plain `.string()`, `.boolean()`, `.array(...)` etc. with no defaults in schemas passed to `generateObject`

### PDF Parsing (`pdf2json` v4)
- Constructor: `new PDFParser(null, true)` — second arg enables raw text mode
- API: `parseBuffer(buffer)`, `getRawTextContent()`, events `pdfParser_dataReady` / `pdfParser_dataError`
- These APIs are confirmed working in v4.0.2

### AI SDK Versions
- `ai` v6 + `@ai-sdk/openai` v3 are in use — these are the current installed versions, not a mistake

### LaTeX Live Preview (SwiftLaTeX)
The live PDF preview compiles LaTeX in the browser using SwiftLaTeX (pdfTeX compiled to WASM).

**Engine chain:** `LatexPdfPreview` (React) → `/latex-worker.js` (Web Worker) → `/PdfTeXEngine.js` → `/swiftlatexpdftex.js` (inner Worker) → `/swiftlatexpdftex.wasm`

**Rendering:** After compilation, `components/latex-pdf-preview.tsx` renders the resulting PDF using `react-pdf` (`Document` + `Page` components with `renderTextLayer={false}` and `renderAnnotationLayer={false}`) — raw canvas output, no browser PDF viewer chrome.

**TeX Live bundle** is served as static files from `public/texlive/pdftex/` with one subdirectory per kpse format number:
- `3/` — TFM font metrics (requested **without** extension — kpse format 3 implies `.tfm`)
- `10/` — Format file: `swiftlatexpdftex.fmt` (14 MB, pre-compiled by the WASM itself)
- `11/` — Font maps: `pdftex.map`
- `26/` — LaTeX packages/classes/configs (`.sty`, `.cls`, `.def`, `.cfg`, `.tex`)
- `32/` — Type1 fonts (`.pfb`, requested **with** extension)

**Critical gotchas:**
- The format file (`pdftex/10/swiftlatexpdftex.fmt`) **must** be generated by `scripts/generate-swiftlatex-fmt.mjs` using the actual `swiftlatexpdftex.wasm`. A format compiled by a different pdfTeX binary will fail with "Fatal format file error; I'm stymied" due to mismatched dump magic numbers.
- `swiftlatexpdftex.js` uses synchronous XHR to fetch TeX files. The `fileid` response header is used as the in-WASM filename; since Next.js static file server doesn't set this header, `swiftlatexpdftex.js` has been patched to fall back to extracting the filename from the URL path.
- For `\input{name}` without extension (e.g. `\input{glyphtounicode}`), pdfTeX passes the bare name to kpse. `swiftlatexpdftex.js` has been patched to retry with `.tex` appended when format 26 returns a non-200. Additionally, `glyphtounicode` (no extension) is served alongside `glyphtounicode.tex`.
- Do **not** use `scripts/bundle-texlive.sh` — it copies a locally-compiled pdflatex format which is incompatible with the WASM.

**Regenerating the format file** (only needed if `swiftlatexpdftex.wasm` changes):
```bash
node scripts/generate-swiftlatex-fmt.mjs
# Outputs: public/texlive/pdftex/10/swiftlatexpdftex.fmt (~14 MB)
```

## Architecture

**Ascendume** is an AI-powered resume builder. Users create/edit resumes, get AI-enhanced bullet points, and export to PDF.

### Route Structure

The `app/` directory uses two route groups plus standalone routes:

- `app/(marketing)/` — Landing page (unauthenticated)
- `app/(application)/` — Dashboard (authenticated); page components live here as `-page.tsx` files rather than in `page.tsx` directly
- `app/(application)/[id]/` — Resume builder for an existing resume (URL: `/<id>`)
- `app/(application)/new/` — Resume builder for a new resume (URL: `/new`)
- `app/(application)/layout.tsx` — Auth-guarded layout: redirects unauthenticated users to `/auth/sign-in`, wraps builder routes with `DashboardLayoutClient` + `DashboardSidebar`
- `app/auth/sign-in/` and `app/auth/sign-up/` — Auth pages
- `app/api/` — Route handlers:
  - `resumes/` — CRUD for resumes
  - `resumes/[id]/export/` — PDF export
  - `resumes/parse/` — Parse uploaded PDF resumes
  - `ai/enhance/` — AI bullet point enhancement (Vercel AI SDK streaming)
  - `auth/[...all]/` — better-auth catch-all handler

### Data Layer

- Schema: `lib/db/schema.ts` — tables: `user`, `session`, `account`, `verification` (better-auth managed), `resumes`, `personalInfo`, `workExperience`, `education`, `skills`, `projects`, `certifications`
- DB client: `lib/db/index.ts` using Neon serverless driver
- Auth: `lib/auth.ts` (server), `lib/auth-client.ts` (client)

### Resume State

`lib/resume-context.tsx` provides a React context (`ResumeContext`) with the full resume data shape (`ResumeData`) used throughout the builder. The builder page reads/writes through this context, which syncs to the DB via `lib/use-resume-persistence.ts`.

### Key Conventions

- **Server Components by default** — add `"use client"` only for interactivity/hooks
- **Absolute imports** — always use `@/components/...`, `@/lib/...` etc.
- **Named exports** everywhere except `page.tsx`, `layout.tsx`, `route.ts`
- **`cn()`** from `@/lib/utils` for conditional Tailwind classes
- **Drizzle inferred types** — use `typeof table.$inferSelect` for DB model types
- **No default exports** outside Next.js conventions
- Schema changes require both `npx drizzle-kit generate` + `npx drizzle-kit push`
